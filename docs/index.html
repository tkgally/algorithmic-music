algorithmic-music

An experiment in making genuinely good instrumental music with code in the browser — plain HTML, CSS, and JavaScript with the Web Audio API. No samples, no libraries, no build step, no server: every note is synthesized live, and each engine runs entirely offline from a single folder.

The project is a long-running collaboration between Tom Gally, who directs it and judges the music, and Claude (Anthropic's AI), which does all of the research, writing, and engineering across autonomous sessions. Its bet: browser music generators are easy to build but usually musically disappointing, and the cure is knowledge plus iteration — a serious, cited knowledge base about how music works and why people enjoy it, driving engine designs that improve through evaluation and listener feedback. Everything here — this page, the engines, and the knowledge base — is written and built by Claude, and dedicated to the public domain (CC0).

The engines

Each engine is a self-contained, deliberately different generative instrument. Pick one and press play; every piece is reproducible from a seed and shareable by link.

ENGINE 01

Tonal Classical

A short piece in classical style, built to actually finish: a theme (a parallel period that asks and answers with real cadences), a contrasting middle, a varied return of the theme at the climax, and a closing coda that accelerates into a final cadence and a lengthened last chord. Harmony is generated backward from its cadences; melody is shaped contour-first around pinned goal tones. Major or minor, any key, adjustable tempo and reverb.

More engines coming

The plan is a family of deliberately contrasting engines across different genres and architectures — ambient/generative, groove-based, and non-Western tuning systems among them — each built from the same knowledge base. This catalog will grow; engines are numbered so they are easy to reference and compare.

How it works

  • Knowledge base. Claude researches music theory, the psychology of listening, genre traditions, composition craft, algorithmic techniques, Web Audio implementation, and evaluation — every page ending in concrete implications for engines, every claim cited.
  • Engines. Contrasting generators built from that knowledge, published here on GitHub Pages. Each is fully transparent about being AI-designed, deterministic from a seed, and downloadable to run offline.
  • Improvement loop. Listener feedback and computational metrics feed findings back into the knowledge base, and the engines get better over time.

Source, knowledge base, and session-by-session history: github.com/tkgally/algorithmic-music. All original work is dedicated to the public domain under CC0 1.0. Built with vanilla HTML/CSS/JS and the Web Audio API — no dependencies.
