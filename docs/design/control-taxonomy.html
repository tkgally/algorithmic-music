Control taxonomy Draft for review

The first design document for the comprehensive site: the concrete controls exposed at each of the three tiers — Start, Intermediate, Advanced — with names, ranges, defaults, and what each one does; plus a proposed shortlist of Start-level genres. Written by Claude for Tom's asynchronous review.

How to read this / how to give feedback. This is a proposal, not a built feature — nothing here is final until you've reacted to it. The quickest things to weigh in on are collected as “For Tom” boxes throughout and gathered again in §9. React however is easiest — edit this file, drop notes in wiki/project-open-questions.md, or say them in a session prompt; a later session folds your decisions into the wiki and starts building. Once you sign off, the settled taxonomy graduates from this review doc into the wiki (control-surfaces-and-user-parameters.md).

1 · Purpose & scope

The site will compose and perform instrumental music across many styles — familiar ones documented in the wiki and original ones invented on the fly — behind an interface with three levels of control (comprehensive-site vision). This document proposes exactly what those controls are.

It decides: the Start-level genre shortlist; the concrete control at each tier (name, type, range, default, and a one-line mapping into the composition/synthesis pipeline); and the rules every control obeys.

It defers (to later design docs, so this one stays reviewable):

  • the style-pack / style-vector schema — the internal data every control writes into (next design doc);
  • the compact URL encoding of seed + settings (design lives in wiki/music-representation-and-notation.md);
  • the original-style sampling & coherence rules (grounded in wiki/style-invention-and-style-space.md);
  • the actual build (Phase 3c).

Basis: control-surfaces-and-user-parameters.md (precedent + HCI evidence) and style-invention-and-style-space.md (the axis space), plus the five preliminary engines as proven-control precedent.

2 · Design principles

Seven rules, drawn from the HCI and generative-music evidence, that every control in this document obeys. They are why the tiers can share one engine.

  1. Auto = the seed decides. Every control defaults to auto. An untouched control is not “off” — it means let the seed choose a value within the selected genre. Setting a control pins it (removing it from what the seed samples). This is the project's own precedent (the percussion engine's per-axis auto), and it makes Start a special case of Advanced — almost everything on auto — rather than a separate product.
  2. Every control is a deterministic input. (version, seed, all settings) → piece, exactly and repeatably. A control is a value fed to the seeded pipeline (genre → sample/invent the style → compose → perform → synthesize), never a post-hoc tweak of the audio.
  3. Tiers are progressive disclosure, not different engines. Start ≈ layer 1 (a few consequential choices), Intermediate ≈ the familiar musical parameters, Advanced ≈ the full board including invented parameters. Moving up a tier reveals more controls without discarding what you set; moving down hides them but keeps their (auto or pinned) values.
  4. Label for perception; scale for perception. Start/Intermediate controls use high-consensus listener words (brightness, warmth, energy, density, space), not engineering names. Numeric controls step in perceptual units: dB for loudness, semitones/octaves for pitch, ratio steps for tempo (a JND is ~5%, so equal-BPM steps feel coarse when slow and trivial when fast). Technical/original names appear only at Advanced, glossed on first use.
  5. Defaults are the product. Because most controls stay on auto, what “auto” produces is what most listeners hear. Defaults must be tasteful, not merely safe: a minimal surface raises the bar on the sampler behind it. Every genre ships a good-sounding default region.
  6. Randomize is first-class and constrained. A big Play and a dice/“new piece” control sample only the un-pinned axes, within the chosen genre — “surprise me here,” not “surprise me anywhere.” Because the seed and settings live in the URL and copy to the clipboard, a lucky result is always recoverable and shareable, so exploration is lossless.
  7. Guard the failure modes. The one-knob macros and the small Start set must still yield coherent, non-arbitrary music. Behind every simplified control sits the coherence machinery (a stable reference pitch, within-piece repetition, a novelty budget); simplicity at the surface, not in the generator.

Counts are soft targets. The HCI evidence says the win comes from strong defaults + progressive disclosure, not from starving the Advanced tier — so Start stays small (first-timers lack articulated preferences) while Advanced is deliberately generous.

4 · Start tier

The flow: the user picks one genre, optionally nudges a couple of headline dials, and presses Play. A random seed that encodes those choices is generated and the piece plays; the seed fills in everything else. The dice reseeds within the same genre; the seed is shown, editable, and copyable.

6 · Advanced tier

The full board — everything above at finer resolution, the style-space axes exposed directly, per-layer control, and the parameters invented for this site. The choice-overload caveat runs in reverse here: the expert user wants all of it, so this tier is deliberately generous and is not capped for its own sake.

6.2 · Per-layer overrides

Pin any setting per voice/role rather than globally — e.g. this palette for the lead, that register for the bass, more swing on the drums only — generalizing the engines' per-axis pinning to per-layer.

7 · Auto, seed & the URL

  • What Play does: collect the chosen genre + any pinned controls → generate a seed that encodes them → the seed samples every un-pinned axis within the genre's region → compose → perform → synthesize. Same (version, seed, settings) ⇒ same piece, always.
  • The URL: seed + settings ride as a compact, versioned suffix on the URL; the same full URL reproduces the piece exactly. A version field lets old links keep working as the site grows. Encoding details are deferred to music-representation-and-notation.md; it need not be human-decipherable.
  • The dice (“New piece”): reseeds only the un-pinned axes — so pinning three controls and hitting the dice explores just the rest, within the genre.
  • Copy: the seed copies to the clipboard; a lucky accident is never lost.
  • Crossing tiers keeps your settings: revealing Advanced never resets Start; hiding it never discards a pin.

8 · How controls map to the engine

One pipeline serves all three tiers because the tiers differ only in how many axes the user pins vs. lets the seed sample:

  • A Start genre is a preset region of the style space — a style-vector center plus per-axis spreads. Picking it sets the sampling region; the seed draws a point inside.
  • An Intermediate control is a friendly handle on one axis (or a small rollup of a few), shifting the region's center or narrowing its spread on that axis.
  • An Advanced control is direct access to a raw axis, a per-layer pin, or an invented parameter (arc, blend, novelty budget…).

The object all of them write into is the style vector — the subject of the next design doc, the style-pack / style-vector schema. That doc will fix each axis's storage type and the coherence constraints that keep a sampled point listenable; this document only fixes what the user sees and can set.

  1. The Start genre shortlist (§4.1) — is the set of eight right? Add / drop / rename / reorder?
  2. Start size (§4.3) — five headline dials + Genre, or strip Start to Genre + Play only and push the dials to Intermediate?

  5. Invented parameters (§6.3) — right first set, and should any be surfaced earlier than Advanced?
  6. Should the Start tier ever offer invented styles, or only presets — reserving on-the-fly invention for Intermediate/Advanced? (An open question carried from style-invention-and-style-space.md.)
  7. Tiers as panels or modes? One screen that progressively discloses, or three distinct modes you switch between (settings preserved either way)?
