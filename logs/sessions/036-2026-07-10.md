## The nine requests and what was done

1. **Intermediate palette dropdown: descriptions + 5 more sets.** Each palette now carries a `desc`, shown in the dropdown as "Name ‚Äî description" (the dropdown is now built dynamically from the *current style's* palettes, honoring `dynamicValues:'palettes'`, which the previous static labels ignored). Five **shared generic palettes** (`EXTRA_PALETTES` in `style.js` ‚Äî Bowed strings / Reed & breath / Glass & bells / Electric / Mallets & plucks) are appended to every genre, so each genre now has **8‚Äì9 palettes** (3‚Äì4 authored + 5 generic). **Auto sampling draws only among a genre's own authored palettes** (kept genre-appropriate); the generic sets are opt-in.
2. **Advanced: replace the palette dropdown with instrument checkboxes.** A new **`checkset`** control type (`instruments`): a grid of 24 instrument checkboxes (grouped Melodic / Low / Percussion), checked = in the current composition. Backed by a bitmask over a frozen 24-voice `MASTER_INSTRUMENTS` list; `apply()` rebuilds `v.ensemble` to exactly the checked voices (kept voices retain their register/level; added voices get a role from a voice‚Üírole table). `effectiveEnsemble` skips the palette remap once the mask is set. The palette dropdown is **hidden in Advanced** (replaced by the checkboxes); it stays in Intermediate. **Reliability note:** because strategies resolve parts with `ens.find(role)` over a small role vocabulary, *unchecking* always mutes; *checking* a new instrument only sounds if it fills a role the strategy voices (an honest limit of role-driven composition).
3. **More Advanced parameters.** Five new controls mapping onto vector fields the engine already reads but nothing exposed: **Micro-timing** (`laidBack`), **Rubato** (`rubato`), **Chord rate** (`harmonicRhythm`, 1/4-bar ‚Ä¶ 4/bar), **Melodic motion** (`grammar.stepBias`), **Melodic range** (`grammar.range`).
4. **Reset button** (right of Continuous, all modes): returns every pinned control to auto (keeps the current style + seed; re-plans from here if playing). `AMApp.resetControls`.
5. **Continuous play: ~1 s gap between pieces.** `advancePiece()` now starts the next conductor at `ctx.currentTime + 1.0` (was +0.03), so the outgoing piece's tail rings out into a short breath before the next begins.
6. **Playhead visibility.** The piano-roll playhead is now **centered** (window `t0 = t ‚àí span/2`, was 0.7), **wider** (3 px √ó DPR), and **high-contrast** (a coral line + soft halo that reads on both light/dark themes).
7. **Minimum length ~60 s.** `LENGTH_SECS[0]` 65‚Üí60; the invented-style and per-genre auto length minimums lowered (e.g. lofi 80‚Üí60, classical/folk 95‚Üí65, ambient 150‚Üí95) so many pieces can be auditioned quickly, especially in continuous play. Medium/long targets kept.
8. **Structure window** below the viz (all modes): "what this piece is doing", generalized from Engine 04. A static description from the vector (key ¬∑ meter/tempo ¬∑ harmony ¬∑ arc ‚Üí ending ¬∑ instruments; meld/invented specifics; "listen for" signature hints) plus a **live section strip** built from the conductor's `timelineUnits` (consecutive units grouped by section, current one highlighted). Because composition is just-in-time, the strip grows as the piece plays. Invented styles and melds surface their own section labels/signatures, so it is more varied than any single engine's window.
9. **Invent a style in Start.** The ‚ú¶ Invent-a-style button is now visible in every mode (the Start-only guard that forced it off was removed).

## Serialization: the V2 URL layout (the load-bearing infrastructure change)

Requests 1‚Äì3 all needed the compact URL to carry new/wider fields, so `serialize.js` gained a **V2 frozen layout** (append-only per the design's versioning rule): the 30 V1 controls (with `palette` widened 2‚Üí4 bits) plus `laidBack`/`rubato`/`harmonicRhythm`/`stepBias`/`melRange` and the 24-bit `instruments` mask. `encode` writes the current version; `decode` dispatches on the 6-bit version tag to the matching frozen table, so **old V1 links still decode** (their palette is 2-bit, the new controls default to auto). The live control registry mirrors V2 (a dev assert checks id+bits equality); V1 stays a frozen historical constant. Empty payload is still **14 chars**; everything-pinned is **36**.

## Decisions (working assumptions, Tom can override)

1. **Auto keeps genre-appropriate palettes; generics are opt-in.** Preventing a random cross-genre recolor (e.g. a bowed lead on a percussion piece) unless the user chooses it.
2. **The instrument checkboxes rebuild the ensemble to exactly the checked set.** Predictable ("the piece uses these instruments"); the cost is that adding is bounded by what the strategy voices. Documented in the findings page.
3. **Continuous gap is ~1 s of tail-ring, not hard silence** ‚Äî a musical breath rather than a dead stop.
4. **Length minimums lowered per genre by judgment** (not a uniform cut): shorter where a genre tolerates it (lofi/electronic/classical/folk), gentler where length carries the form (ambient/jazz). Easy to retune from Tom's ear.
