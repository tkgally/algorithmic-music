### The document

Wrote **`docs/design/control-taxonomy.html`** — a styled, self-contained review document (matches the site's visual system; renders on GitHub Pages with no dependencies). Contents:

1. **Purpose & scope** — decides the per-tier controls + the Start-genre shortlist; explicitly defers the style-pack schema, the compact URL encoding, the coherence rules, and the build to later docs/sessions so this one stays reviewable.
2. **Seven design principles** distilled from `control-surfaces-and-user-parameters.md`: (1) auto = the seed decides; (2) every control is a deterministic input to `(version, seed, settings) → piece`; (3) tiers are progressive disclosure, not different engines (settings persist across tiers); (4) perceptual labels + perceptual scaling (dB, semitones, ratio-step tempo); (5) defaults are the product; (6) randomize is first-class and constrained to the current genre; (7) guard the failure modes behind the simplified surface.
3. **The three tiers at a glance** — a table (who, control count, mental model, real-world analogue).
4. **Start tier** — the flow (pick a genre, optionally nudge ~4 dials, Play); the **proposed 8-genre shortlist** (grounded table); and the Start controls (Genre picker, Energy, Mood, Length, Space, Play/New-piece, Seed).
5. **Intermediate tier** — a **~20-control** table grouped into Time & Feel / Pitch & Harmony / Texture & Density / Sound & Space / Form & Variation / Expression, each row with type, range/values, default (auto), and the pipeline target it maps to.
6. **Advanced tier** — the **13 style-vector axes** exposed directly (from `style-invention-and-style-space.md`), **per-layer overrides**, and **8 parameters invented for the site** (intensity arc, development mode, timeline type incl. 7. **Auto / seed / URL model** — how Play works, the dice, the copyable seed, cross-tier persistence.
8. **How controls map to the engine** — Start genre = a preset style-space region; Intermediate control = a friendly handle on one axis; Advanced = raw axis / per-layer pin / invented knob. All write into the style vector (the next design doc's subject).
9. **Nine open questions for Tom**, with seven "For Tom" callout boxes inline at the relevant sections; the Start shortlist is flagged as the one to weigh in on first.

## Decisions & notes

- **HTML, not Markdown, for review docs.** GitHub Pages serves HTML deterministically (Markdown rendering depends on Jekyll config); HTML also matches how the rest of `docs/` works and lets the doc be styled for readability. Established `docs/design/` as the home for review docs.
- **No new wiki page yet.** The taxonomy is a *proposal for review*, not a settled decision, so per the reconciliation recorded in session 029 it lives only in `docs/` for now; once Tom signs off, its settled parts fold into `control-surfaces-and-user-parameters.md` (or a new wiki page). This keeps CLAUDE.md rule 4 satisfied without prematurely canonizing an unreviewed proposal.
- **Counts are proposals.** Start ~5 dials, Intermediate ~20 — both flagged as soft and put to Tom (should Start be just Genre + Play?; which of repetition-amount / section-count / tension-curve / complexity earn an Intermediate slot?).
