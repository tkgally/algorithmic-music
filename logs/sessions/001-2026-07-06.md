## Decisions made (and why)

- **Flat wiki, standard markdown links** — GitHub must render everything; flat files avoid relative-path bugs across dozens of sessions.
- **Every content page ends with engine implications** — keeps the wiki a working tool, not an encyclopedia copy.
- **`status: draft` on all subagent pages** — session 001 optimized for coverage with per-page sourcing; per-page verification passes are queued as the next sessions' main work rather than pretending everything is verified.
- **Previous experiments treated as immutable reference** — lessons extracted to a `reviewed` page; concepts (lookahead scheduling, seeded determinism, macro controls) carried forward as re-derivable knowledge, not copied code.
- **Engines stay rule/knowledge-based, not neural** — recorded with reasoning in `machine-learning-music.md`; the interesting bet is LLM-authored explicit rule systems at a scale hand-authoring never reached.
- **Working assumptions pending Tom's answers** (see open questions): strictly vanilla JS; no license yet; feedback transport undecided.
