# Session 047 — 2026-07-13

**Model:** Claude Opus 4.8
**Scope:** One small, Tom-directed change — rename the first control-level mode from **Start** to **Basic** on the comprehensive site, revising every function name, variable name, and reference to *this mode* accordingly.

## What Tom asked

> The three modes selected at the top of the main page are START / INTERMEDIATE / ADVANCED. "START" will be confusing — people might think it is the control to start playback. Rename this mode to BASIC, and make sure function names, variable names, and references to this mode are all revised accordingly.

## Why

Affordance collision. The three control tiers sit at the top of the page as *Start · Intermediate · Advanced*; "Start" is also the verb of the transport (the ▶ Play button begins playback), so a first-time listener could mistake the tier selector for the play control. "Basic" keeps the plain-language ladder (Basic → Intermediate → Advanced) with no overlap against a nearby action verb.

## What changed (all in the live product + its code/gate)

A pure relabel — the tier is only a **rendering-column selector** (it decides which of the three `.ctlTier` columns a control renders into); it is **never serialized**, so URLs, seeds, and determinism are completely untouched.

- **`docs/index.html`** — the mode button label `Start` → `Basic`; the container id `startControls` → `basicControls`.
- **`docs/app.js`** — in `buildControls`: `startWrap` → `basicWrap`, `$('startControls')` → `$('basicControls')`, and the tier tests `c.tier === 'start'` / `c.tier !== 'start'` → `'basic'`; the "added to Start" comment → "added to Basic"; `SITE_VERSION` `1.0.1` → **`1.0.2`** (patch, user-visible label change).
- **`docs/lib/style.js`** — the four base-tier control definitions (`energy`, `mood`, `length`, `space`) `tier: 'start'` → `tier: 'basic'`, plus the registry comment (`tier: 'start'|'int'|'adv'` → `'basic'|…`).
- **`docs/about.html`** — the operating-instructions prose: "Start / Intermediate / Advanced" → "Basic / …" and "Start has a handful of headline dials" → "Basic has …".
- **`experiments/tools/site-smoke.mjs`** — two mode-referring test labels (`invent-a-style visible in Start` → `… in Basic`, and the batch-2 comment).

The mode buttons are wired by **numeric index** (`mode0/1/2`, `state.uiMode` 0|1|2, `setMode(m)`), so no button id, event handler, or the URL `uiMode` field changed.

## Deliberately NOT renamed (flagged for Tom)

- **"Start genres"** — the eight preset styles (classical, ambient, lofi, jazz, folk, electronic, cinematic, percussion) are called the "Start genres" throughout the wiki, logs, and `CLAUDE.md`. This is a *distinct concept* from the control tier (it means "the presets available from the base tier on"), and renaming it would ripple project-wide well beyond Tom's request. Left as-is. If Tom wants this term changed too, it's a separate, larger pass.
- **`docs/design/*.html`** — the Phase-3b design docs still say "Start" for the tier. They are dated review snapshots that already diverge from the shipped UI in other ways (e.g. they still describe multi-select genre melds, made single-select in session 035), so they read as historical record, not living UI docs. Left unedited.
