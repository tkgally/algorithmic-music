## Result — percussion → 0.2.0

### (1) Sliding scales — a parametric generator replacing three fixed styles

The v1 composer had three `composeX` functions, each with a fixed six-section `FORMS` arc — which is exactly why the three ensemble types looked/sounded structurally similar (all rose to a climax and fell). v2 replaces them with **one parametric generator** driven by a **recipe sampled from the seed** (`sampleRecipe`), over these axes:

- **development** (0..1): cyclic (loop core, vary edges) ↔ through-composed.
- **formType**: cyclic / additive / cellDev / throughComposed / process / moment.
- **arc**: the intensity curve, chosen from **nine** shapes (`rise`, `arch`, `archLate`, `terraced`, `wave`, `johakyu`, `frontload`, `flat`/moment, `swellcut`) — the biggest variety lever, since a percussion piece's form *is* its intensity curve (`arcAt(name, x)`).
- **timeline**: clave / bell / Euclidean / **Xenakis sieve** (residue classes) / pulse-only / none.
- **density**, **looseness**, **leadActivity**, **callResponse**, **interlock**, **pitched**.

One unified `generate()` realizes any recipe: the timeline (the inviolable clock-key), a low anchor, interlocking Euclidean/sieve ostinati that **enter as the section intensity rises** (additive layering), cell development for developmental recipes (rotation/displacement/augmentation/diminution across the ensemble), an improvising lead, color accents (metal/bell/clap/chime), textures (shaker/scrape), fills at 4/8/16-bar boundaries, unison hits at high intensity, and a gong at the peak downbeat and the coda ending. The three named genres survive as **presets** (`REGIONS`) that bias the recipe toward a region; **`auto`** (the new default) samples the whole space; and **every axis is optionally user-overridable** (the composer's `ov()` treats null/'auto'/'' as "sample it").

Measured over 24 `auto` seeds: **8 distinct arc shapes, 5 timeline types, all 6 form types, 24 distinct ensembles** — genuine variety, not three templates. A bug found and fixed along the way: an explicit `development` override was being clamped by the form-type correlation (now the correlation only applies when the user hasn't pinned development).

### (2) Four more voices, each a different type + seed-varied ensembles

Four new **original** voices in `synth.js` (additive), each a distinct gesture-type:

- **`clap`** — a hand-clap / stick: a comb of 3–4 band-passed (~1 kHz) noise bursts a few ms apart + a body tail (the "spread hands"); a `stick` tag = one sharp click. A body-percussion **impact** type (enables hand-clap ensembles).
- **`scrape`** — a guiro/cabasa **rasp**: band-passed noise gated by a fast (~40–60 Hz) square tremolo (the ridges). A *scraped* gesture, distinct from the shaker's single burst.
- **`chime`** — a clear, tonal ringing **pitched metal** (steelpan/hang/crotale spirit): a near-harmonic modal bank (1 : 2 : 3 : 4, lightly stretched) + a strike shimmer, long ring. Distinct from the clangy `metal` and the wooden `mallet`; usable as a second melodic color.
- **`friction`** — a cuíca-style **friction drum**: a sawtooth through two vowel-formant band-passes with a pitch-bend glide (up or `down`) + a friction squeak. The organic, voice-like member.

The palette is now **11 voices**. A new **ensemble sampler** (`sampleEnsemble`) draws voices into functional role-slots (anchor / timeline / interlocking mids / colors / textures / lead / melodic) from the catalog `CAT` via `ROLE_POOL`, so each seed gets a different *combination* — all-metal (the Xenakis *Métaux* ensemble), skins-and-wood, hands-and-shakers, minimal trios, etc. The user can pin the set with an **Ensemble** preset (auto / skinsWood / allMetal / handsShaker / malletMetal); a fix ensures preset pools don't leak a default voice when a small pool is exhausted. Stereo pans are spread deterministically.

### Performer + UI

The performer's looseness now reads the recipe's continuous **looseness** axis (not three discrete styles), the four new voices got laid-back offsets and bus routing (clap/friction → percMid; scrape/chime → percHi). The UI adds a **Style** selector with **auto** default, an **Instruments** preset, per-axis override controls (Development / Density / Looseness / Melody sliders each with an "auto" checkbox default-on; Arc / Timeline / Form selects with an "auto" option), a twelve-lane visualization with labels, and a self-report that shows the sampled recipe.

## Key decisions

- **Additive, engine-05-local.** The 4 new voices + helpers are additive to `synth.js`; only `docs/engines/05-percussion/` was re-vendored. Engines 01–04 keep their own older vendored copies → byte-for-byte unaffected.
- **`auto` is the default style** — the seed drives the whole recipe out of the box (Tom: "determined by the seed"); presets and per-axis controls are the optional manual overrides (each defaults to "auto").
- **Deliberately generic, still.** The recipe reaches many structures but the timelines/forms remain *musical technology* (a clave, a bell, a sieve, an arc), not named-tradition reproductions — the Agawu caution (`west-african-rhythm.md`). Named-tradition presets remain a future option.
- **Determinism preserved** (seeded RNG streams for recipe/ensemble/form/generation; fixed ratios + shared seeded noise for the voices) so the offline gate stays stable and browser-identical.
