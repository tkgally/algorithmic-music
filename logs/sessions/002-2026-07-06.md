## Decisions made (and why)

- **Two license files, not one** (`LICENSE` MIT + `LICENSE-wiki` CC0), both at repo root. Keeps GitHub's license detector happy (MIT primary for code), keeps the split unambiguous, and avoids putting a non-`.md` file inside `wiki/` (which would trip the conventions/lint expectations for that directory). CC0 is the right instrument for "public domain" on prose; the file uses the standard CC0 deed statement plus a link to the canonical legalcode rather than pasting ~1.5k words of legal text (a future session can embed the full legalcode if Tom wants it).
- **Dependency nuance kept as a "strong default," not a loophole.** Tom said tiny vendored helpers are OK "if ever genuinely needed"; every edit preserves the default and the `file://`/no-runtime-network constraints, so the bar for an exception stays high.
- **Genre note placed as an italic "Project note" after the lede** on the two genre pages, rather than rewriting the knowledge prose — keeps the pages about the music while making the launch-priority discoverable to future sessions.
- **Daily-site mechanism recorded as dormant-until-Phase-4** everywhere, so current sessions aren't confused into looking for a site that doesn't exist. Flagged the one implementation detail it needs: a stored "last site update" timestamp in repo state.
