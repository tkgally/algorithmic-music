# Session 048 — 2026-07-13

**Model:** Claude Opus 4.8
**Scope:** Tom's substantive request — make **"Invent a style"** pieces more varied and stop them sharing a distinctive mid-piece "break" where notes repeat dronelike for a measure or two. All in the invented-style composer (`docs/lib/invent.js`) + a regression guard.

## What Tom reported

> Playing pieces in "Invent a style" there is often or even usually a "break" section in the middle where notes are repeated dronelike for a measure or two. While such a mid-piece break is fine occasionally, I want the pieces to be varied and to avoid distinctive shared features like this mid-piece break.

Two example seeds:
- `#p=DqPTGsKPIAAAAAA` (seed a8f4c6b0, melodyAccomp, functional, dev 0.452 → the old **arch** form, which had a break)
- `#p=DSrGowaPIAAAAAA` (seed 4ab1a8c1, hocket, functional, dev 0.207 → the old **cyclic** form, which had a break)

## Diagnosis (structural, not a bug)

The session-038 form grammar (`buildSections`) had **exactly one shape per development band** and hard-wired an identical `break` into two of the three:
- `cyclic` (dev < 0.35): `A A′ [break] A″`
- `arch`   (dev < 0.7):  `A B A′ [break] peak`
- `through`(dev ≥ 0.7):  `A B C peak A′` (no break)

`development = r.next()` is uniform, so ~70% of pieces fell in a break-bearing band; net ~**38%** of random invented seeds had a break (chorale/tintinnabuli were already exempt; ~45–50% among the textures that permit one). And `writeBreak` always did the **same** thing — a held low drone + a quiet repeating pulse for a phrase — so every break sounded alike: a shared signature, the opposite of per-seed identity, plus too little form variety (3 fixed shapes over the whole dev range). Confirmed by decoding both of Tom's seeds (both had a break) and a 200-seed sweep.

## The fix (`docs/lib/invent.js`)

1. **Form is DRAWN, not thresholded.** `development` now *biases* a weighted family draw (weights clamp to a 0.1 floor, so every dev value keeps spread), and each family offers **~5 shapes** — **15 skeletons** where there were 3. Nearby-dev seeds can diverge. Every shape still guarantees a stated theme `A`, ≥ 1 recognizable return `A′`/`A″`, and one `peak` (preserves FM3/FM4 + the theme-return test).
2. **The `break` is occasional and optional.** Most shapes take their mid-piece contrast from **real departure sections** (`B`/`C`, off-tonic, new material); only a minority carry a `thin` slot. The length-fit shrink removes the break first, so short (~60 s) pieces — both of Tom's examples — drop it entirely.
3. **When a thin section occurs, its character is DRAWN** (`pickContrast` → `writeContrast`), matched to the ensemble: **breakdown** (percussion groove alone), **solo** (moving lead spotlight, on the contrast contour — never the committed theme), **stripped** (moving lead+bass duet), **pulse** (rhythmic pitched cell on shifting gamut tones), **drone** (the pre-047 hold, now rarest). Static kinds (drone/pulse) shortened to two bars. Each keeps something sounding (no-silence gate) and falls back if its voice is absent.

Renamed `writeBreak` → `contrastDrone` (verbatim body) and added `writeContrast` + `contrastBreakdown/Solo/Stripped/Pulse` + `pickContrast`. **Determinism untouched** — the new draws use the previously-**unused** `form` RNG stream and the per-unit stream, so no other stream shifts; same seed → same piece. As in 037–039, invented *seeds* now compose different (better-formed, more varied) pieces; old invented links still decode and play.

## Measured effect (before → after, over random invented seeds)

| | before | after |
|---|---|---|
| pieces with a mid-piece break | ~38% | **~7–9%** |
| distinct section-sequences / 200 seeds | ~3 | **~65** |
| ways the break is realized | 1 (drone) | **5** (drone now rarest, ~1%) |
| Tom's two example seeds | both had a break | **neither does** (real departures instead) |
