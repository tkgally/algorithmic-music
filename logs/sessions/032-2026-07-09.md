## What was done

### 1. Style-vector schema (`docs/design/style-vector-schema.html`) — new

The second Phase-3b design doc, companion to the control taxonomy. Sections:

- **The model** — a style is a fielded *vector* (center + per-axis spread; a region, not a point); a running piece is that vector walking a *path* over the form (Xenakis book-of-screens / game RTPC); presets, invented styles, and melds differ only in provenance.
- **The schema fields** — ~24 fields in four layers (**material · per-voice behavior · evolution · surface**) plus cross-cutting (signature rules, novelty budget), each typed **categorical / ordinal / continuous** — the type drives both meld-blending and serialization ("interpolate quantities, re-choose symbols," from `meta-composition-and-style-machines.md`).
- **Presets, invented styles, signatures** — presets are hand-tuned points (the 8 Start genres); invented styles are seeded samples + a 1–3-axis novelty budget + 2–3 authored signature rules (EMI-minus-corpus, per Cope); signatures are what make a style *readable* within one piece.
- **The two-genre meld (§5)** — answers Q1. A meld is **per-axis parent selection, not a numeric average**: categorical fields snap to one parent, continuous fields blend; the load-bearing rule (from `hybridization-and-fusion.md`) is that the **rhythmic chassis and the harmony/mode come from *different* parents**, with one committed chassis. **Balance even (50/50), flagged provisional** per Tom. Compatibility gates (tuning↔harmony, meter, feel, groove-vs-rubato) reject/repair clashes; cap at two parents; gradient blend is the default operator; three worked examples.
- **Coherence constraints (§6)** — the cross-axis couplings (tuning↔harmony, tuning↔timbre, register↔spacing, tempo↔density, meter↔groove, form↔duration) plus a parseability gate and the automated critics, applied to any vector before playback.
- **Determinism & the live morph (§7)** — `(version, seed, pinned settings) → vector → per-bar composition`; live changes mutate the vector and the just-in-time composer reads it each bar; the three response speeds map onto the schema layers; the **genre/blend swap is a two-bar crossfade** (Tom), flagged provisional.
- **Serialization & the link (§8)** — seed + only the *pinned* settings → versioned bit-packed base64url (design owned by `music-representation-and-notation.md`); the link is the **destination snapshot only** (Tom).
- **Three shared generators (§9)** — Schillinger interference rhythms, Xenakis sieves, Koenig tendency masks (all deterministic; Engine 05 already ships a sieve), with the Backus anti-rhetoric caveat (generators produce possibility, never value).
- **Decisions & open questions (§10)** — Tom's three calls recorded; Claude's own design calls flagged; open items (preset numbers, signature-bank size, categorical-vs-ordinal fields, trajectory axis count) deferred to later sessions/listening tests.

Built on `style-invention-and-style-space.md`, `meta-composition-and-style-machines.md`, `hybridization-and-fusion.md`, `music-representation-and-notation.md`, and the engines' recipe/params precedent.
