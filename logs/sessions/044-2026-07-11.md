### 3. Font-delivery decision (Tom, via AskUserQuestion)
The one issue with a literal drop-in: the handoff loads **Google Fonts via a CDN `<link>`** — the project's **first-ever runtime external fetch** (the old page fetched nothing). This collides with two documented `CLAUDE.md` hard rules: rule 2 ("no network at runtime; must run from `file://`; first-party libs are vendored, never fetched") and the requirement that `site-smoke.mjs` "must stay green" (the CDN link turns it red offline — 15 identical `net::ERR_CONNECTION_RESET`, all font-load; every functional assertion passed). I surfaced this to Tom rather than silently pick.

**Tom chose: self-host the fonts.** So I:
- Fetched the Google Fonts `css2` for the declared weights (Space Grotesk 400;500;600;700, JetBrains Mono 400;500;700) through the agent HTTPS proxy with a Chrome UA. Both families are **variable fonts** — Google serves **one `latin`-subset woff2 per family** covering the whole weight axis. Vendored just those two files into `docs/fonts/` (`space-grotesk-latin.woff2` 22 KB, `jetbrains-mono-latin.woff2` 31 KB; the `latin` subset covers every glyph the site shows, incl. em-dash, curly quotes, ·, ›, …).
- Added local `@font-face` rules (variable `font-weight: 400 700`, `font-display: swap`) to the top of `docs/css/site.css`; the CSS already references the families by name via `--grot`/`--mono`, so no other CSS change was needed.
- Removed the two `preconnect` links + the stylesheet `<link>` from both `index.html` and `about.html`.

Result: **no runtime network fetch remains**, the site works from `file://`, and the Console typography is preserved exactly.

## Decisions worth remembering

- **Self-hosted fonts over CDN** (Tom): preserves the exact Console typography while honoring the no-runtime-fetch / `file://` rule and keeping the gate green. Both target fonts are variable → one `latin` woff2 per family is the complete, minimal vendor. If a future non-English gloss needs `latin-ext`, add that subset then.
- **Preliminary-tests links dropped from the public site** (Tom): the engines remain as regression baselines but are no longer advertised to listeners. The smoke-gate assertion that required the link was retired to match — updating the gate to the new product truth, not weakening it to pass.
- **The engine was genuinely untouched.** The redesign is presentation-only; every id/selector the engine and gates depend on survives.
