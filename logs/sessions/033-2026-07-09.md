>
> **Your spec — read these first; they are the contract:**
> - `docs/design/site-architecture.html` — **the build blueprint**: folder layout, module list, the just-in-time composition loop, determinism, compact URL, and the **walking-skeleton-first build order (M1–M8)**. Build in that order.
> - `docs/design/control-taxonomy.html` — the controls at each tier (Start/Intermediate/Advanced), the 8-genre Start shortlist, multi-select two-genre melds, the live-change model.
> - `docs/design/style-vector-schema.html` — the style vector the controls write into: ~24 typed fields, presets vs invented styles vs melds, the meld rules, coherence, serialization.
> - Supporting wiki: `engine-architecture.md`, `findings-shared-lib-foundation.md` (the classic-script/`file://` constraint), `shared-libraries.md`, `music-representation-and-notation.md` (URL encoding), the style pillar (`style-invention-and-style-space.md`, `hybridization-and-fusion.md`, `meta-composition-and-style-machines.md`), and the genre pages behind the eight Start genres (Classical→`western-classical-tradition.md`, Ambient→`ambient-and-generative-genre.md`, Lo-fi→`hip-hop-and-beat-making.md`, Jazz→`jazz-and-improvisation.md`, Folk→`european-folk-and-dance.md`, Electronic→`electronic-and-dance.md`, Cinematic→`film-and-game-music.md`, Percussion→`percussion-music.md`).
>
> **Hard requirements (from the docs — do not relax):**
> - Vanilla HTML/CSS/JS + Web Audio only; original first-party libraries loaded as **classic scripts** into one global namespace (NO cross-file ES-module `import` — CORS-blocked under `file://`); no build step, no server, no network at runtime; instrumental only; CC0; the UI is upfront that everything is written and built by Claude.
> - The site lives in **one folder**: `docs/index.html` (the entry point — it **replaces** the current early-landing `docs/index.html`) plus `docs/lib/`, `docs/styles/`, `docs/app.js`, `docs/_selftest.html`, per the architecture doc. Keep a footer link to the preliminary tests (`preliminary-tests/`), which stay live.
> - **Determinism:** `(version, seed, pinned settings) → piece` exactly; live tweaks are the one allowed exception (so the shared link stores the destination snapshot). Seed shown, editable, copy-to-clipboard; compact versioned `base64url` URL.
> - **Three modes** with a selector at the top (settings preserved across modes). Start = ~5 dials (Energy/Mood/Length/Space) + **multi-select Genre (max 2, two blend into a meld)** + Play/dice; Intermediate ~20 familiar controls; Advanced the full board incl. the invented parameters. Every control defaults to `auto` = the seed decides.
> - **Live changes** must morph the piece **without restarting**, via the just-in-time composition loop (compose a bar/phrase ahead of the playhead, reading the current style vector); three response speeds (instant / next-boundary / re-plan); a **two-bar crossfade** for a genre/blend swap; meld balance **even (50/50)** for now.
> - **Do NOT modify the preliminary-test engines** in `docs/preliminary-tests/` (they are regression baselines). Reuse their proven voices/composers/performers/UI by copying and adapting into `docs/lib/`, never by referencing across folders.
>
> **Build approach — walking skeleton first (this is how it fits in one session):**
> - Build **M1–M4 yourself** as the coherent core: skeleton (index + app + vendored `rng/transport/synth/fx`, a Play button that seeds a trivial loop from the URL) → **one genre (Classical) playing end-to-end and deterministically via the JIT loop** → the three-mode UI + compact URL → then widen. Get one genre fully working before anything else; it de-risks the whole pipeline.

> - **M1–M4 (a working, seeded, shareable three-tier site composing all eight genres end-to-end, live-playable) is the target floor.** Land M5 (melds), M6 (live changes), M7 (invented styles), M8 (polish) as far as the session reaches; each is additive, so an unfinished one is a clean follow-up. **A solid partial site beats a broken complete one** — never leave the site non-loading or non-deterministic to cram in more.
>
> **Validation (reuse the engines' harnesses):**
> - An `OfflineAudioContext` render gate per genre: no clip, no gap, level in band, and **determinism — same seed → byte-identical PCM**.
> - A headless **ui-smoke**: the site loads; each Start genre plays; the URL round-trips; a live control change does **not** restart the transport; 0 console errors.
> - Keep `node experiments/tests/run.js` and `node experiments/tools/wiki-lint.mjs` green.
>
> **Budget:** one coherent, working result within ~70% of the context window; prioritize the walking skeleton over breadth.
>
